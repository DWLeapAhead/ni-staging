FAQ

Common Questions

Is Shasta really free? What's the catch? +
Shasta is genuinely free and open source. There is no freemium bait-and-switch. NI offers Shasta as a community tool because we believe every organization deserves basic cloud security visibility. Enterprise features like scheduled scans, team collaboration, and compliance reporting are available in Transilience Enterprise plans.
What permissions does Shasta need on my cloud accounts? +
Shasta requires read-only permissions on your cloud accounts. For AWS: SecurityAudit managed policy. For Azure: Reader role. For GCP: Viewer role. Shasta never writes to, modifies, or deletes any resources in your cloud environment.
Is my cloud data safe with Shasta? +
Shasta collects metadata and configuration data only — never your actual data. All scan data is encrypted in transit and at rest. You can also run Shasta in a fully air-gapped, self-hosted mode using the open-source version on GitHub.
Can Shasta scan AI workloads on non-major clouds? +
Currently Shasta covers AI workloads on AWS SageMaker, Azure AI Studio, Google Vertex AI, and popular self-hosted inference setups. Support for additional platforms is on our public roadmap — contributions welcome via GitHub.
How is Shasta related to Rainier and Kamet? +
Shasta is part of the Transilience platform by NI. When used together with Rainier (AI SOC) and Kamet (AI Pen-Testing), findings from Shasta feed into a unified security intelligence loop — misconfigurations discovered by Shasta become targeted pen-test attack vectors in Kamet and monitored by Rainier in real-time.
